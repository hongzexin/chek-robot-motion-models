# 收录方法 / Collection method

## 目标 / Goal

把“看起来很多”但“真正能判断的不多”的机器人动作资源，整理成一个有筛选价值的索引仓库。

## 基本流程 / Basic workflow

1. 找到公开入口
2. 判断它是模型、代码入口、数据集还是演示页
3. 记录机型、用途、下载方式、部署风险
4. 用统一 schema 填进 `models.json`
5. 同步导出到 `models.csv`
6. 运行校验脚本

## 来源优先级 / Source priority

优先级从高到低：

1. 官方仓库或官方模型页
2. 维护活跃的公开仓库
3. 可验证的直链
4. 二级整理文章或社区帖子

## 重点核验项 / What to verify

- 页面是否真的存在
- 是不是明确给出了文件入口
- 是否需要登录或 gated access
- 适配机型有没有写清
- 更像展示动作还是底座控制
- 是否明显只适合仿真

## 明确不做的事 / What we intentionally do not do

- 未授权镜像第三方权重
- 把营销文案当成可用性证明
- 把数据集和模型混成一类
- 把“项目介绍页”写成“可直接下载”

## 维护建议 / Maintenance notes

- 新条目进入仓库前，至少保留一条风险说明
- 如果链接失效，保留历史记录，但明确标注
- 如果资源可用性变化较大，更新 `last_checked_note`
